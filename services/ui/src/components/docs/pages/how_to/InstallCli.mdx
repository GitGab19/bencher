## Debian

Bencher provides a convenient `deb` file for installing on Debian
and Debain based distributions like Ubuntu.

<pre>
  <code>
    {`wget https://github.com/bencherdev/bencher/releases/download/v${BENCHER_VERSION}/bencher_${BENCHER_VERSION}_amd64.deb
sudo dpkg -i bencher_${BENCHER_VERSION}_amd64.deb`}
  </code>
</pre>

## GitHub Actions

Install the Bencher CLI using the [GitHub Action](https://github.com/marketplace/actions/bencher-cli).
See [how to use GitHub Actions](https://bencher.dev/docs/how-to/github-actions) for more details.

<pre>
  <code>
    {`- uses: bencherdev/bencher@v${BENCHER_VERSION}`}
  </code>
</pre>

There is an optional `version` argument to specify an exact version of the Bencher CLI to use.
Otherwise, it will default to using the latest CLI version.

<pre>
  <code>
    {`- uses: bencherdev/bencher@main
  with:
    version: ${BENCHER_VERSION}`}
  </code>
</pre>


These two commands are functionally equivalent.
However, the first protects you from any breaking changes in the Bencher CLI GitHub action by using a tagged release.
Therefore, the former is the suggested method (ex: <code>{`- uses: bencherdev/bencher@v${BENCHER_VERSION}`}</code>)

## From Source

For all other environments, the suggested install method is from source.

<pre>
  <code>
    {`git clone --branch v${BENCHER_VERSION} https://github.com/bencherdev/bencher.git
cd bencher
cargo install --path services/cli --locked`}
  </code>
</pre>

## Other Linux Distributions

A standalone Linx CLI binary is also available for download
if you know what you are doing.

<pre>
  <code>
    {`wget https://github.com/bencherdev/bencher/releases/download/v${BENCHER_VERSION}/bencher`}
  </code>
</pre>
